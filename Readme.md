# 本小组课题 : 简易_康复训练动作_计数器

## 零、   使用说明
按照步骤依次运行即可;  
1. 使用文件夹 *1* 中的 **camera.py** 调用设备摄像头,  
   程序运行后,在 *stdin* 中连续输入**字符** *s* 即可连续拍照,一个s字符对应一张照片,  
   输入**字符** *q* 即可退出程序;  
2. 运行文件夹 *1* 中的 **main.exe** 即可对之前的拍照内容进行分割和标记坐标输出,  
   支持最多三个通道的像素标记,具体的标记颜色值可根据需要在文件夹 *1* 中的 **macros.h** 中进行修改;  
   (具体的说,就是修改该MARK(可选R,G或者B)对应的三个standardxx 宏)  照片取色器可在本文件夹下的*PHOTOTOOLS*中找到;
   输出文件名称格式为 **count.md** *总数据文件*, **countX.md** *X标记对应的数据文件*;
3. 运行文件夹 *2* 中的 **data_r.exe** 即可对上述原始数据文件进行分割,得到下一步需要的紧凑形式数据文件;
   输出文件名称格式为**output_X.md** *X标记对应数据文件*;
4. 在上位机中运行文件夹 *3* 中的 **analysis.m** 文件, 可进行简单的周期性动作分析;
   具体包括*动作的次数*和动作对应的*时间-坐标图*;
   可根据动作的具体周期对**analysis.m**文档中的*波谷间最小间隔*进行调整;

## 一、   概念层&配套硬件声明
1. 主要功能  
  通过被动型动作捕捉对测试者的简单周期动作进行识别和计数(可通过按钮设置周期/次数/读取或清理记录);

2. 硬件配置  
  * 摄像单元 : Raspberry Pi Camera Module V2 IMX219PQ;
  * 存储单元 ：大小至少*16GB*以上的U盘;
  * 分割单元 : Raspberry Pi 4B 1GB;
  * 分析单元 : 搭载*MATLAB*的上位机;
3. 开发平台  
    **Anaconda 4.7.11**  
    **VScode 1.40.2**    
    **MATLAB 2017**  
4. 示例动作  
   * 右手竖直向上举过头顶并完全伸直;
   * 右手竖直向右挥臂,直至肩肘腕三关节共线并且整只手臂水平于地面;
## 二、   功能层声明
1. 初始化  
每个记录周期开始时,需要要求  
    ***被测者右手***

   * 朝向右侧;
   * 完全伸直;
   * 水平于地面; 
1. 动作捕捉  
		开始计数后,对摄像头回传的照片进行分析,找出标记的三个关节所在位置,并记录;  
3. 动作识别  
		利用上述记录进行三关节运动轨迹分析,给出正误判断;  
4. 输出信号  
		如果病人动作符合预设要求,语音提示一个动作完成;  
5. 最终结果

  3. 问题1:关于流程的设计问题;  
   - 抛弃方案: 直接读取至CPU并进行分析;
     - 原因: 对CPU使用频繁;
   - 抛弃方案: 使用DMA直接存入SD卡之后再采样;
     - 原因: 传输和采样都是占用的SD卡的带宽,没有使用DMA的意义;
   - 现用方案:  实时压缩存储+实时分析;
     - CPU直接读取数据,  
     - 但每读两个字节(RGB565基本像素单元)使用一个字节进行位标记,存在内存中;
     - 标记满一张图就暂停采样,将数据存入SD卡;
     - 每采样两张图就计算一次一阶中心距, 
       并在内存中的轨迹图上更新标记,  
       之后实时通过语音调整被测者姿势;  
       ***(需要考虑采样过快导致的提醒频繁问题)***
     - 每做完一次动作就将轨迹图存入SD卡,  
       判定轨迹图是否符合标准,  
       符合就语音输出  **动作成功**;  
       不符合就语音输出**动作失败,请重新来过**,失败次数加一;  
       之后清空内存中的轨迹图;  
     - 循环**预设次数**+**失败次数**;
     - 当整个循环结束以后,  
       依据**失败次数/成功次数**输出本次循环评价,  
       并将评价存入SD卡;  
       之后语音提示结束本次训练;
     - 当整个周期结束以后,还需要根据多次循环评价给出周期评价;  
    ***至此,上周的最后一个问题"如何将图片存入SD卡"已在概念层解决;***
